# 2者間ゲーム実験仕様書（動的更新版）

## 1. 目的
2者間ゲーム（囚人のジレンマ、スタグハント、チキンゲーム、およびベースラインとしてのハーモニーゲーム）における意思決定と反応時間（RT）を測定する実験プログラムの作成。
最終的には以前作成したEEM（jsPsychタスク）のコードと統合し、Qualtrics上で動作させる（埋め込む）ことを想定。

## 2. 開発ワークフロー（進捗ステータス）
- [x] **ステップ1:** ローカルでの基本構造作成（`experiment.html` / `main.js` / `style.css`の作成）
- [ ] **ステップ2:** Qualtricsへの実装準備（データ保存処理等）
- [ ] **ステップ3:** 既存課題（EEM）との統合

## 3.1 タスクフロー
1. 教示 (ルールの説明、ポイントの表の見方)
2. 理解度クイズ (2問：自身のポイント、相手のポイントの読み取り)
3. 練習試行 (3回)
4. 本番試行（46項目 × 2ブロック・キーボード）
5. 終了画面
6. フルスクリーン解除
7. Qualtrics移行処理

## 4. 各要素の詳細要件

### 4.1 教示画面
* 基本的に**すべてマウスのクリック操作（ボタン）**で進行する。
* 「協力」「非協力」など価値判断を含むラベルは絶対に使用しない。
* 教示の冒頭で、「実験で獲得した最終的なポイントに応じて、参加報酬とは別に追加のボーナス報酬が支払われる」旨を明確に説明する。
* 教示の前半で実験概要、後半で「ポイントの表」を画面に出して以下を説明：
  * あなたと相手の選択が交差したマスが結果となる
  * 自分の選択によって相手のポイントが変わり、相手の選択によって自分のポイントが変わる
  * それぞれの選択によるポイントの変化の具体例を、ハイライトを用いて1ページずつ論理的かつ具体的に説明する。
  * 相手は事後マッチングによって特定の1人に決まること。
  * ほかのプレイヤーは完全にランダムで匿名であり、これ以降の関わりがないこと。
  * フィードバック（相手が何を選んだか、最終的な利得はいくつか）は提示されないこと。

### 4.2 クイズ
* キーボード入力（本番）と混同しないよう、**マウスのクリック（画面上のボタン）**で解答させる。
* 計2問実施する。
  * Q1: 「あなたの利得」はどちらか？（正解：左側の数字）
  * Q2: 「相手の利得」はどちらか？（正解：右側の数字）
* **ループ制限:** 間違えた場合は再挑戦ループに戻るが、**各問題最大2回まで**とする。2回間違えた場合は正解を提示して強制的に次に進める。

### 4.3 試行フロー・時間パラメータ
* クイズ終了後、本番のキーボード操作環境を強制するため**マウスカーソルを非表示**にする。
* 練習課題の開始画面で「これ以降はキーボード（F/Jキー）を使用する」旨を明確に説明し、スペースキーで練習試行を開始させる。
1. **注視点 (Fixation):** 1000ms
2. **ブランク画面:** 500ms
3. **ポイントの表の提示（キー入力待ち）:**
   * 画面中央にマトリックス形式のポイントの表を提示。
   * あなたの選択: 「[F]キー」「[J]キー」
   * 相手の選択: 「パターン1」「パターン2」
   * 表内の数値にはすべて「pt」の単位を付与する。
   * キーボードの「F」または「J」が入力されるまで待機（制限時間なし）。
4. **選択のハイライト（500ms）:**
   * 選択した列（Fなら左、Jなら右）の背景色を薄い青色（#e7f1ff 等）に変更し、500ms提示する。
5. **試行間ブランク:** 500ms

### 4.4 利得表のUIと視覚多様性への配慮
* 数値の色付けに依存しないよう、各セル内で**「あなた」「相手」というラベルの直下に黒字で数値を表示**する形式を採用し、視覚的多様性に完全対応する。
* 13インチモニターでも見やすいよう、**全体の幅を960px程度に拡大**し、文字サイズを大きく確保。
* **画面中央への完全配置:** 表の左側ヘッダー（パターン1など）をCSSで表枠外の絶対配置にすることで、純粋なデータセルの境界線の交差点が完全な画面中央（X:50%, Y:50%）に来るよう配置する。

## 5. 項目水準（条件）とデータ構造

* **ゲームの種類:**
  * 囚人のジレンマ (PD)
  * スタグハント (SH)
  * チキンゲーム (CH)
  * ハーモニーゲーム (HG: アテンションチェック用)

* **データ構造と計算ロジック:**
  数値をベタ書きせず、以下の「ベース×倍率」のロジックで利得表を動的に生成する。
  1. 最小単位のベース配列： `const base = [8, 6, 4, 2]` （被験者が暗算しやすいように変更）
  2. 以下の14パターンの倍率ペアを定義し、ジレンマゲーム3種に適用する（14ペア × 3種 = 計42試行）
     - 対称・低ステークス (1): (1, 1)
     - 対称・高ステークス (1): (10, 10)
     - 非対称・被験者が有利 (6): (4, 3), (3, 2), (2, 1), (3, 1), (5, 1), (10, 1)
     - 非対称・被験者が不利 (6): (3, 4), (2, 3), (1, 2), (1, 3), (1, 5), (1, 10)
  3. HG用に以下の4パターンの倍率ペアを追加する（計4試行）
     - (1, 1), (10, 10), (10, 1), (1, 10)
  4. 生成した配列（v1 > v2 > v3 > v4）のインデックスを、ゲームの種類に応じてT, R, P, Sにマッピングする。
     - PD: T=v1, R=v2, P=v3, S=v4
     - SH: R=v1, T=v2, P=v3, S=v4
     - CH: T=v1, R=v2, S=v3, P=v4
     - HG: R=v1, T=v2, S=v3, P=v4 (相互協力が双方の最大利益かつ強支配戦略)
  5. ブロック構造
     - ジレンマ42試行 ＋ HG4試行 = 1ブロック計46試行。これを `jsPsych.randomization.repeat` 等で2ブロック繰り返す。
